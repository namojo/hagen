# 하겐 (Hagen) — 거버넌스 내장 하네스 팩토리

> One trigger, governed team. — **h**arness + **agen**t
> 도메인 문장 하나로 에이전트 팀을 생성하고, 그 자리에서 거버넌스까지 자동 장착한다.

지능은 모델이 낸다. 방향은 하겐가 잡는다.

## 무엇이 다른가

기존 하네스 도구들은 둘 중 하나였다:
팀을 잘 찍어내지만 그 팀이 운영될 규율은 안 심어주거나(팩토리형),
규율은 완벽한데 팀 생성이 수동이거나(거버넌스형).

하겐는 **생성 시점에 거버넌스를 주입**한다. "하네스 구성해줘" 한마디에:

1. 도메인 분석 → 6개 아키텍처 패턴 중 팀 구조 설계
2. 에이전트 정의(.claude/agents/) + 스킬(.claude/skills/) 생성
3. **거버넌스 자동 장착** — 헌법, 세션 재주입 훅, 프로파일별 게이트, 감사 로그
4. 기계 검증(validate.ts) + 품질 게이트(7/10) 통과 후 인수인계

## 설치

```
/plugin marketplace add namojo/hagen
/plugin install hagen@hagen
```

## 사용

Claude Code에서:

```
하네스 구성해줘. 기술 블로그 운영 팀이 필요해 —
주제 리서치, 초안 작성, 기술 검수, SEO 최적화를 하고 서로 검토했으면 좋겠어.
```

## 거버넌스 프로파일

| 프로파일 | 핵심 | 대상 |
|----------|------|------|
| light | 기록은 남긴다 | 콘텐츠·리서치 |
| standard | 완료는 증거로 증명한다 | 사내 자동화·파이프라인 (기본) |
| strict | 되돌릴 수 있어야 진행한다 | 프로덕션·배포·결제 |

위험도를 보고 자동 추천하며, 변경은 한 줄이다:
`bun scripts/inject-governance.ts --profile strict`

## 설계 원칙

- **LLM/스크립트 분업** — 판단은 LLM, 설치·검증은 TypeScript. LLM이 안 해도 되는 일을 LLM에게 시키지 않는다.
- **파일 기반 핸드오프** — 모든 인수인계는 송장 붙은 파일. 감사 가능하고, 세션이 죽어도 남는다.
- **비례 거버넌스** — 게이트는 변경 성격에 따라 라우팅된다. 모든 변경에 모든 게이트를 걸면 게이트는 소음이 된다.
- **증거 기반 완료** — "완료했습니다"는 주장이다. 테스트 통과와 산출 파일이 증거다.
- **기록→학습 루프** — _workspace/는 쌓는 곳이 아니라 회고로 하네스 자체를 개선하는 재료다.

## 계보와 감사

이 프로젝트는 다음의 개념에서 배웠다 (코드 미포함, 개념 참조):
revfactory/harness의 팀-아키텍처 팩토리 모델, gstack의 운영 가드레일과 스킬 체이닝,
superpowers의 세션 훅 강제와 2단계 검수, ai-workspace-standards의 거버넌스 계층 발상.
모든 템플릿·문서·코드는 본 저장소에서 새로 저술되었다.
